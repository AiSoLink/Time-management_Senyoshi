# 内部設計書（Next.js + FastAPI ローカルWeb）
版：2.0  作成日：2026-02-16 09:37:28

---

## 1. 実行形態（ローカル）
- Frontend：Next.js（localhost:3000）
- Backend：FastAPI（localhost:8000）
- 保存：ローカルディスク（app_root/work/）

---

## 2. ディレクトリ（実行時）
- `companies/`：会社×機種プリセット（JSON）
- `work/jobs/<jobId>/`
  - `input/`：アップロードPDF
  - `output/`：excel/log/skipped
  - `state.json`：status/進捗/集計
  - `events.log`：進捗イベント（任意）

---

## 3. ジョブ実行モデル
- フロント：ジョブ作成→jobId受領→ポーリングで状態監視
- バック：ジョブ作成後、バックグラウンドで処理（BackgroundTasks/スレッド）
- 進捗：state.jsonに随時書き込み（ダウンロードの可否もここで管理）

---

## 4. PDF抽出エンジン（Python）
- pdfplumber：ヘッダー抽出（運行ID/車両/営業所/メーター/得点/ヘッダー時刻）
- camelot（lattice）：明細抽出（出庫/帰庫/待機/休憩）
- 日時正規化：前行より時刻が戻ったら翌日扱い
- 作業時間補完：到着/出発があれば差分で算出（分）
- 集約：1運行=1行（分割後列は空欄）

---

## 5. ローカル前提のセキュリティ
- CORS：localhostのみ
- アップロードサイズ上限（例：合計200MB、単体50MB）
- 会社名/ファイル名のパス安全化（トラバーサル防止）
