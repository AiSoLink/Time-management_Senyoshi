# PDF→Excel正規化 ローカルWebアプリ 仕様書（Next.js + FastAPI）
版：2.0（ローカルWebアプリ化）  
作成日：2026-02-16 09:37:28（JST）  

---

## 1. 目的
運行日報PDF（機種：mimamori / telecom）を**ローカルPCで処理**し、運行ID（`ID-xxxxx`）単位で **1運行=1行** に集約したExcel（A〜AM固定ヘッダー）を出力する。  
UIはWebアプリとして操作できる（会社→機種→PDFアップロード→処理→ダウンロード）。

- **PDFは外部に送信しない（ローカル完結）**
- 抽出ルールは会社×機種のプリセットJSON（フェーズ1は開発側が編集）
- “分割後”列やランキング計算は後段のExcelマクロで実施（本処理は空欄）

---

## 2. スコープ

### 2.1 フェーズ1（実装対象）
- フロント：Next.js（React + TypeScript）
- バック：FastAPI（Python）
- プリセット：ローカルJSON（`companies/<会社名>/{mimamori,telecom}.json`）
- PDF投入：Web UIで複数PDFアップロード（ドラッグ&ドロップ）
- 実行：アップロードされたPDFを一括処理（途中停止なし）
- 失敗PDFはスキップしログを残す
- 出力：Excel（.xlsx） + ログCSV をダウンロード可能

### 2.2 フェーズ2（将来）
- 初回のみ割付UI（プリセットビルダー）でJSONをGUI編集・保存

### 2.3 非スコープ（フェーズ1）
- クラウド同期、DB保存、複数端末共有
- 抽出ルールをユーザーが自由編集（BPO運用のため）

---

## 3. 主要要件（機能）
- 会社一覧表示（companiesディレクトリを読み取り）
- 会社追加（会社名入力→companies/<会社名>/作成、雛形JSON生成）
- 機種選択（mimamori/telecom）
- PDF複数アップロード（同一運行IDが複数PDFに跨っても結合）
- ジョブ実行（進捗表示、完了通知）
- 結果ダウンロード（Excel、ログCSV、失敗PDF一覧）

---

## 4. 非機能要件
- ローカル完結（ネットワーク不要で利用可能）
- 失敗PDFが混じっても処理継続（スキップ）
- 会社/機種が増えてもJSON追加で運用可能
- Windows優先（ARM含む）でも成立する開発・実行構成（Python venv + Node）
